## 2. 목표 (Goals)

1. **메커닉당 대표 스테이지 1~2개를 선정**하여 비주얼/게임플레이 퀄리티를 memory-dodge 수준으로 끌어올린다
2. **네온 획일화 탈피** — 각 대표 스테이지에 고유한 색상 팔레트와 비주얼 정체성 부여
3. **코드 다이어트** — 장식 렌더링 80% 제거, 타겟 600~900줄
4. **HUD 간소화** — 게임 중 표시 정보 3~4개로 축소
5. **미선정 스테이지 정리** — 레지스트리에서 제거 (코드는 보존 가능)

## 3. Non-Goals

- 독립 게임(memory-dodge 이전)의 수정
- 새로운 메커닉 추가
- create_stage.js 생성 파이프라인 자체의 리팩토링
- 갤러리/플레이 페이지 UI 변경

## 4. 구현 계획

### Phase 1: 선정 (Selection) — 메커닉당 대표 스테이지 선정

각 메커닉에서 가장 독특한 모디파이어를 가진 1~2개를 선정한다.

**선정 기준:**
- 모디파이어가 핵심 루프를 가장 많이 변형하는 것
- 비주얼 테마가 차별화 가능한 장소
- 코드 복잡도 대비 게임플레이 깊이가 높은 것

**예비 선정안 (pulse-fencing 기준, 다른 메커닉도 동일 방식):**

| 순위 | 스테이지 | 선정 이유 |
|------|---------|----------|
| 1 | paper-temple-pulse-fencing-shrink | 유일한 반경 이동 + 시간 제한, 가장 차별화됨 |
| 2 | midnight-harbor-pulse-fencing-tempo | 리듬 메커닉, 완전히 다른 체감 |
| 제거 | 나머지 8개 | 기본 루프에서 크게 벗어나지 못함 |

### Phase 2: 비주얼 리디자인 (Visual Redesign)

선정된 스테이지에 적용할 비주얼 가이드라인:

| 항목 | Before (현재) | After (목표) |
|------|-------------|-------------|
| 액센트 색상 | 5~6색 네온 | 2~3색 (warm + cool + danger) |
| 배경 렌더링 | 30~60줄 장식 코드 | 단일 그라디언트 + 최소 앰비언트 1개 |
| shadowBlur | 8~15개 요소 | 2~3개 요소 (플레이어, 위험 영역) |
| 오버레이 카드 | 그라디언트 보더 + 글래스모피즘 | 단색 카드 + 컬러 보더 (memory-dodge 패턴) |
| 폰트 스타일 | 과도한 letter-spacing, uppercase | 기본 폰트, 2~3가지 크기/굵기 |
| 고유 정체성 | 네온 glow | 장소별 고유 비주얼 요소 (건물, 자연물 등) |

### Phase 3: HUD/UX 간소화

| 항목 | Before | After |
|------|--------|-------|
| HUD 값 | 4~8개 패널 | 3~4개 (핵심 지표만) |
| 컨트롤 설명 | 게임 중 표시 | 메뉴에서만 표시 |
| 힌트 패널 | 상시 표시 | 제거 또는 첫 플레이만 |
| 메뉴 카드 | 과도한 그리드/뱃지 | 제목 + 설명 + 시작 버튼 |

### Phase 4: 코드 다이어트

- drawBackdrop() 함수 80% 축소
- 미사용 장식 함수 제거
- 타겟: 스테이지당 600~900줄

### Phase 5: 미선정 스테이지 정리

- registry.js에서 미선정 스테이지 제거
- 디렉토리는 `community-stages/_archived/`로 이동 (또는 삭제)
- 갤러리에서 미노출
